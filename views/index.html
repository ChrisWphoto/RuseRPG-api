Route Documentation & Examples

/users/:id
- Takes: url parameter e.g. users/1234
- Returns: Array of 1 JSON object representing all user profile data.

/lookUpCardio
- Takes: void
- Returns: Array of JSON objects representing all cardio workouts on file

/lookUpStrength
- Takes: void
- Returns: Array of JSON objects representing all strength workouts on file

/postUser
- Takes: {user_id: 5, userName: 'Tony', email: 'm@m.com'}
- Returns: Array of 1 JSON object representing all user profile data.
- Note: if user does NOT exist, new user will be created and default profile will be returned

/postWorkout
- Takes: {workout_id : int, fk_user_id : TwitterID}
- Returns: JSON Confirmation of Success or Error
- Note: Timestamp is added as current time by default

/postCardio
- Takes: {duration_seconds : int, fk_workout_id : int, fk_workout_user_id : TwitterID, fk_lookupcardio_id : int }
- Returns: JSON Confirmation of Success or Error
- Note: cardio_id is auto generated

/postStrength
- Takes: {reps : int, fk_workout_id : int, fk_workout_user_id : TwitterID, fk_lookupstrength_id : int }
- Returns: JSON Confirmation of Success or Error
- Note: strength_id is auto generated
